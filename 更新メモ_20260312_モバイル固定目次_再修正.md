# 更新メモ（2026-03-12 / モバイル固定目次 再修正）

## 対象
- `index.html`

## 再修正内容
- モバイル固定目次の表示条件を強化
  - 画面幅メディアクエリだけでなく、JavaScriptで端末条件を判定して表示を制御
  - 条件: `max-width:1024px` または `hover:none + pointer:coarse`
  - 条件一致時は `mobile-rich-toc.enabled` と `body.has-mobile-rich-toc` を付与して、固定目次を強制表示
- 既存上部目次（`toc-section`）は `body.has-mobile-rich-toc` 時に非表示化
- 下部固定目次と本文が重ならないよう下余白をクラス制御に統一

## 目的
- 端末やブラウザ差異で下部メニューが非表示になるケースを回避し、スマホで確実に目次を表示する
